How do we read source code?

Take part in our study!

Source code is often written and then left untouched for long periods of time. It must be written in such a way that a programmer can navigate it even after a long period of time and make necessary changes. Writing code that can be understood without context is quite difficult, however.

Programmers often have a gut feeling about whether code is good or bad. But can this feeling be measured and expressed in numbers? To explore this question, we developed this study.
